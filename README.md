Product Dissection – Swiggy

📌 Project Overview
This project is a comprehensive data architecture study of Swiggy, India’s leading on-demand convenience platform. It reverse-engineers the schema design to understand how data architecture drives the platform's effectiveness

📊 Database Schema Entities:
The system is designed with seven core entities to manage high-volume, real-time operations:
User Entity: Manages profiles, addresses, and registration data.
Restaurant Entity: Tracks location, cuisine types, and ratings.
Menu Entity: Links specific food items and prices to restaurants.
Order Entity: The central hub connecting users, restaurants, and delivery status.
Delivery Partner Entity: Manages logistics, vehicle types, and availability.
Payment Entity: Tracks transaction modes and success status.
Review Entity: Ensures a feedback loop for quality control.
